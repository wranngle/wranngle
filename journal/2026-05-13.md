# 2026-05-13

## Shipped the n8n workflow diff renderer

Built `n8n-diff` — a CLI that takes two exported workflow JSONs and prints
a node-level diff (added / removed / modified nodes, plus connection
changes) with stable ordering so the output is suitable for code review.
Wired the renderer to the n8n CI matrix so PRs touching `workflows/*.json`
get a comment with the diff inline.

- Sorting key: `(node.type, node.name)` — name-only sort drifted when nodes
  were renamed; (type, name) survives renames because the type rarely
  changes for a given role in the graph.
- Connection diff: hash of `(source, sourceOutput, target, targetInput)`
  tuples; resilient to noop reordering inside `connections[]`.
